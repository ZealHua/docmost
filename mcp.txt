  MCP Feature Summary

  The mcp.txt file contains a commit (60848ea9037c2edd8ab90bce11c215a5d6cc9a50) that introduces Model Context Protocol (MCP) support to Docmost. This is
  an enterprise feature that allows AI assistants and tools to interact with workspace content through a standardized protocol.

  Key Changes:

  1. Frontend Changes (apps/client)

  New Components:
   - apps/client/src/ee/ai/components/mcp-settings.tsx - New MCP settings component with:
     - Toggle to enable/disable MCP server
     - Display of MCP server URL (/api/mcp)
     - List of supported tools (page operations, space operations, comments, etc.)
     - Enterprise-only restriction alerts

  UI Updates:
   - apps/client/src/ee/ai/pages/ai-settings.tsx - Added tabs to separate AI and MCP settings
   - apps/client/src/App.tsx - Added route for /ai/mcp
   - apps/client/src/components/settings/settings-sidebar.tsx - Updated label from "AI settings" to "AI"
   - apps/client/src/ee/api-key/pages/user-api-keys.tsx - Added MCP-specific alerts and documentation links

  Zod Migration:
   - Major change: Upgraded from zod v3.25.76 to v4.3.6
   - Updated all form validations from zodResolver to zod4Resolver
   - Updated imports from zod to zod/v4
   - This affects ~20+ form components across authentication, security, settings, and editor features

  Type Updates:
   - apps/client/src/features/workspace/types/workspace.types.ts - Added mcpEnabled?: boolean to workspace and AI settings interfaces
   - apps/client/src/ee/audit/types/audit.types.ts - Added optional avatarUrl to actor type
   - apps/client/src/components/ui/custom-avatar.tsx - Made avatarUrl optional

  Localization:
   - Added 19 new translation keys for MCP features in en-US/translation.json

  2. Backend Changes (apps/server)

  New Dependencies:
   - Added @modelcontextprotocol/sdk v1.27.1
   - Added zod v4.3.6

  Workspace Updates:
   - apps/server/src/core/workspace/dto/update-workspace.dto.ts - Added mcpEnabled boolean field
   - apps/server/src/core/workspace/services/workspace.service.ts - Added MCP settings update logic with audit trail
   - apps/server/src/core/search/search.module.ts - Exported SearchService for MCP usage

  EE Module Update:
   - Updated apps/server/src/ee git submodule commit

  3. Supported MCP Tools

  The MCP server exposes these tools for AI assistants:
   - Pages: search_pages, get_page, create_page, update_page, list_pages, list_child_pages, duplicate_page, copy_page_to_space, move_page,
     move_page_to_space
   - Spaces: get_space, list_spaces, create_space, update_space
   - Comments: get_comments, create_comment, update_comment
   - Attachments & Users: search_attachments, list_workspace_members, get_current_user

  4. Authentication & Integration

   - MCP server endpoint: /api/mcp
   - Authentication via API keys (managed in account settings)
   - Enterprise-only feature with upgrade prompts

  What This Brings:

   1. AI Integration: Enables external AI assistants (like Claude, ChatGPT, or custom agents) to interact with Docmost workspaces through the MCP protocol
   2. Standardized Interface: Provides a consistent, protocol-based way for AI tools to read and manipulate workspace content
   3. Enterprise Feature: Monetization opportunity - MCP is locked behind enterprise edition
   4. Enhanced API: Extends Docmost's API capabilities with a modern, AI-friendly protocol
   5. Tool Ecosystem: Allows integration with the growing MCP ecosystem of AI tools and assistants

  Current State Comparison:

  Not in current codebase:
   - MCP server implementation (backend /api/mcp endpoint)
   - MCP settings UI components
   - MCP-related workspace settings
   - Zod v4 migration (currently on v3.25.76)
   - MCP SDK dependency

  Already present in current codebase:
   - Basic AI infrastructure
   - API key management
   - Enterprise feature framework
   - Settings UI patterns

✦ The changes represent a significant addition to Docmost's AI capabilities, positioning it as an AI-native workspace platform that can integrate with
  modern AI tools and assistants.
